## 1 功能特性

### 1.1 首頁視圖

- **月曆視圖**：按月展示日記，日期格內顯示縮圖，點擊日期查看當天條目
- **列表視圖**：時間線式列表，按日期分組（今天、昨天、往年）
- **那年今日**：展示往年同月同日的所有日記，橫向卡片版面
- **日記卡片**：標題、日期、正文摘要、圖片（支援 1~5+ 張多種版面）
- **統計列**（可選）：連續紀錄天數、總字數、寫日記天數

### 1.2 編輯頁

- **手記式圖片版面**：預設資料夾內的筆記，在 Live Preview 模式下，圖片按首頁卡片樣式排列（1/2/3/4/5+ 張多種版面）
- **超 5 張自動拆分**：同一段連續圖片超過 5 張時，自動拆成多個畫廊，保證每張圖都可見
- **即時更新**：新增/刪除圖片時自動重新渲染
- **圖片刪除**：每張圖支援右上角刪除按鈕

### 1.3 其他

- **多語言**：繁體中文、English、日本語、简体中文
- **虛擬化列表**：基於 @tanstack/react-virtual，長列表效能優化
- **檔案系統監聽**：新建、修改、刪除、重新命名時自動重新整理
- **可設定日期欄位**：支援 frontmatter 中 `date`、`Date`、`created`、`created_time` 等，或自訂欄位

## 2 安裝

### 使用 BRAT（推薦）

1. 安裝 [BRAT](https://github.com/TfTHacker/obsidian42-brat) 外掛程式
2. 在 BRAT 設定中點擊「Add Beta plugin」
3. 填入本外掛程式的 GitHub 倉庫 URL
4. 安裝並在 Obsidian 中啟用

> **Tip**：BRAT 會自動檢查更新並提醒新版本。

### 手動安裝

1. 從 [Releases](https://github.com/你的用戶名/obsidian-journal-react/releases) 下載 `main.js`、`manifest.json`、`styles.css`
2. 放入 `{vault}/.obsidian/plugins/obsidian-journal-react/`
3. 在 Obsidian 中啟用外掛程式

### 自行建置

```bash
cd .obsidian/plugins/obsidian-journal-react
npm install
npm run build
```

建置產物會輸出到外掛程式目錄。

## 3 使用

### 開啟視圖

- **指令**：`Ctrl/Cmd + P` → 輸入「開啟日記視圖」→ Enter
- 或透過指令面板執行「開啟日記視圖」

### 新建筆記

- 在視圖右上角點擊 **+** 按鈕
- 使用設定的預設範本建立當日筆記

### 切換視圖模式

- 點擊月曆/列表圖示，在月曆視圖與列表視圖間切換

## 4 設定

| 設定項 | 說明 |
|--------|------|
| **預設資料夾** | 日記視圖預設開啟該資料夾，編輯頁圖片版面僅在此資料夾內的筆記生效 |
| **日期欄位** | 指定 frontmatter 中的日期欄位，如 `date`、`created`；若無則使用檔案建立時間 |
| **預設範本** | 新建筆記時使用的範本，支援 `{{date}}`、`{{year}}`、`{{month}}`、`{{day}}`、`{{title}}` |
| **編輯頁手記式圖片版面** | 在 Live Preview 中，預設資料夾內筆記是否啟用圖片版面 |
| **圖片間距** | 圖片容器之間的間距（0–30px） |
| **圖片顯示限制** | 每個卡片最多顯示的圖片數量（預設 3，首頁實際以 5 張版面展示） |
| **開啟筆記方式** | 新分頁 / 目前分頁 |
| **顯示統計列** | 是否在頂部展示連續天數、字數等統計 |
| **IndexedDB 儲存用量** | 在維護區域查看條目快取與縮圖的儲存用量，支援一鍵清除 |

## 5 日期與圖片規則

### 5.1 日期提取

1. **優先 1**：從 frontmatter 讀取日期（支援自訂欄位或預設 `date`、`Date`、`created`、`created_time`）
2. **優先 2**：使用檔案建立時間 `ctime`

### 5.2 圖片版面

**首頁卡片**

| 圖片數量 | 版面 |
|----------|------|
| 1 張 | 單列 2:1 |
| 2 張 | 左右各 2:1 |
| 3 張 | 左大 + 右 2 小 |
| 4 張 | 左大 + 右 3 小 |
| 5+ 張 | 左大 + 右 4 小，超出拆成多個畫廊 |

**編輯頁**：與首頁版面一致，超過 5 張連續圖片自動拆成多個畫廊，每張圖支援右上角刪除。

### 5.3 縮圖快取

首頁卡片中的圖片會生成 WebP 縮圖，以加快捲動與載入。

**何時生成？**

| 你的操作 | 系統行為 |
|----------|----------|
| 開啟日記視圖 | 只讀既有快取，不生成 |
| 捲動看到某張圖 | 有快取則直接顯示；沒有則先顯示原圖，背景生成後自動切換為縮圖 |
| 新建筆記後返回 | 新筆記的圖片在列表中顯示時，會依需生成並快取 |
| 修改了某張圖片 | 視為新圖，下次查看時重新生成 |
| 長時間不用某些圖 | 超出配額時，自動淘汰最近最少使用的縮圖 |
| 清除快取 | 清空所有縮圖，下次查看時重新生成 |

**儲存配額與淘汰**

- **IndexedDB**：縮圖總容量上限約 200 MB，超出時按「最近最少使用」自動淘汰
- **記憶體**：最多快取 200 張，同樣採用 LRU 淘汰
- 在「設定 → 維護」中可查看目前用量並手動清除

## 6 隱私與儲存

- **本機儲存**：外掛程式使用 IndexedDB 快取條目元資料與縮圖，資料僅儲存在本機，不會上傳
- **儲存配額**：縮圖快取上限約 200 MB，超出時自動淘汰最近最少使用的項目（詳見 5.3）
- **可清除**：在「設定 → 維護」中可查看儲存用量並一鍵清除快取
- **無網路請求**：外掛程式不向外部伺服器發送任何資料
