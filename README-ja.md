Markdown ファイルをカレンダーで整理し、ジャーナル風のビューを表示。リスト、カレンダー、この日の出来事、画像レイアウトをワンストップで。

## 1 機能

### 1.1 ホームビュー

- **カレンダー表示**：月別にエントリを表示、日付セルにサムネイル
- **リスト表示**：タイムライン形式、今日・昨日・過去でグループ化
- **この日の出来事**：過去の同じ日付のエントリを表示
- **ジャーナルカード**：タイトル、日付、概要、画像（1〜5+ 枚レイアウト対応）
- **統計バー**（任意）：連続日数、文字数、記録日数

### 1.2 エディタ

- **ジャーナル風画像レイアウト**：Live Preview でデフォルトフォルダ内のノートの画像をジャーナルカードと同じレイアウトで表示
- **自動分割**：5枚超の連続画像は複数ギャラリーに分割
- **リアルタイム更新**：画像の追加・削除で即時反映
- **画像削除**：各画像に削除ボタン

### 1.3 その他

- **多言語**：English、简体中文、日本語、繁體中文
- **仮想化リスト**：@tanstack/react-virtual でスムーズなスクロール
- **ファイル監視**：作成・編集・削除・リネームで自動更新
- **日付フィールド設定**：`date`、`Date`、`created`、`created_time` またはカスタム対応

## 2 インストール

### BRAT（推奨）

1. [BRAT](https://github.com/TfTHacker/obsidian42-brat) プラグインをインストール
2. BRAT 設定で「Add Beta plugin」をクリック
3. このプラグインの GitHub リポジトリ URL を入力
4. インストールして Obsidian で有効化

> **Tip**：BRAT が更新をチェックし、通知します。

### 手動インストール

1. [Releases](https://github.com/your-username/obsidian-journal-react/releases) から `main.js`、`manifest.json`、`styles.css` をダウンロード
2. `{vault}/.obsidian/plugins/obsidian-journal-react/` に配置
3. Obsidian でプラグインを有効化

### ビルド

```bash
cd .obsidian/plugins/obsidian-journal-react
npm install
npm run build
```

## 3 使い方

### ビューを開く

- **コマンド**：`Ctrl/Cmd + P` → 「ジャーナルビューを開く」と入力 → Enter

### 新規ノート

- 右上の **+** ボタンでデフォルトテンプレートから当日のノートを作成

### 表示切替

- カレンダー/リストアイコンで切替

## 4 設定

| 設定項目 | 説明 |
|----------|------|
| **デフォルトフォルダ** | ジャーナルビューで開くフォルダ；エディタ画像レイアウトはここ内のノートにのみ適用 |
| **日付フィールド** | frontmatter の日付用フィールド（例：`date`、`created`） |
| **デフォルトテンプレート** | 新規ノート用；変数：`{{date}}`、`{{year}}`、`{{month}}`、`{{day}}`、`{{title}}` |
| **エディタ画像レイアウト** | Live Preview でジャーナル風画像レイアウトを有効化 |
| **画像の間隔** | 画像コンテナの間隔（0–30px） |
| **画像表示制限** | カードあたりの最大画像数（デフォルト 3；レイアウトは 5 枚まで対応） |
| **ノートの開き方** | 新規タブ / 現在のタブ |
| **統計バーを表示** | 連続日数、文字数等を表示 |
| **IndexedDB ストレージ** | メンテナンスでキャッシュを確認・クリア |

## 5 日付・画像ルール

### 5.1 日付抽出

1. **優先 1**：frontmatter の日付（カスタムまたは `date`、`Date`、`created`、`created_time`）
2. **優先 2**：ファイル作成日時 `ctime`

### 5.2 画像レイアウト

| 枚数 | レイアウト |
|------|------------|
| 1 | 単列 2:1 |
| 2 | 左右各 2:1 |
| 3 | 左大 + 右 2 小 |
| 4 | 左大 + 右 3 小 |
| 5+ | 左大 + 右 4 小；超過分は複数ギャラリーへ |

**エディタ**：ホームと同じ；5枚超は複数ギャラリーに分割；各画像に削除ボタン。

### 5.3 サムネイルキャッシュ

ホームカードは WebP サムネイルで高速スクロール。

**いつ生成？**

| 操作 | 動作 |
|------|------|
| ジャーナルビューを開く | 既存キャッシュを読むのみ |
| 画像をスクロールで表示 | キャッシュがあれば表示；なければ原図→バックグラウンドで生成 |
| 新規ノート後に戻る | 新規ノートの画像は表示時にオンデマンドで生成 |
| 画像を編集 | 新規扱い；次回表示時に再生成 |
| 長く使っていない画像 | クォータ超過で LRU 削除 |
| キャッシュをクリア | 全削除；次回表示時に再生成 |

**ストレージクォータ**

- **IndexedDB**：約 200 MB 上限；超過で LRU 削除
- **メモリ**：最大 200 枚；LRU 削除
- **設定 → メンテナンス**で使用量確認・クリア

---

## 6 プライバシー・ストレージ

- **ローカルのみ**：IndexedDB にエントリ・サムネイルをキャッシュ；アップロードなし
- **クォータ**：サムネイル約 200 MB 上限；古いものから削除（5.3 参照）
- **クリア**：設定 → メンテナンスで確認・クリア
- **ネットワーク不使用**：外部サーバーへのリクエストなし
